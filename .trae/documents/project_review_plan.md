# Django + Vue CMS 项目全面审查计划

## 审查目标

对这个 Django REST Framework + Vue 3 的内容管理系统进行全面审查，识别潜在问题、安全风险、性能瓶颈和改进机会。

## 审查方法

### 后端审查
1. 检查 Django settings.py 配置
2. 审查 models.py 数据模型设计
3. 检查 views.py 和 serializers.py
4. 审查权限和认证机制
5. 检查中间件和工具类
6. 审查数据库迁移文件

### 前端审查
1. 检查项目结构和组织
2. 审查组件设计和复用性
3. 检查状态管理（Pinia）
4. 审查路由配置和守卫
5. 检查 API 调用和错误处理
6. 审查样式和 UI 组件

### 安全审查
1. 检查敏感配置（SECRET_KEY, DEBUG）
2. 审查用户输入验证
3. 检查文件上传验证
4. 审查 API 权限控制
5. 检查 CORS 配置

### 性能审查
1. 分析数据库查询（N+1 问题）
2. 检查索引配置
3. 审查缓存使用
4. 检查前端资源优化

## 具体审查步骤

### 步骤 1: 安全配置审查
- 检查 `SECRET_KEY` 是否硬编码或有默认值
- 检查 `DEBUG` 模式配置
- 审查 CORS 配置是否过于宽松
- 检查数据库配置安全性
- 审查 JWT 配置（token 有效期、刷新机制）

### 步骤 2: 数据库模型审查
- 检查模型字段定义和约束
- 审查外键关系和级联删除
- 检查数据库索引配置
- 审查数据迁移文件
- 检查 N+1 查询问题

### 步骤 3: API 和权限审查
- 检查 API 端点的权限控制
- 审查用户输入验证
- 检查文件上传验证（类型、大小）
- 审查序列化器的数据清洗
- 检查敏感数据是否泄露

### 步骤 4: 前端代码审查
- 检查组件设计和复用性
- 审查状态管理逻辑
- 检查路由守卫实现
- 审查 API 错误处理
- 检查 XSS 防护

### 步骤 5: 性能优化审查
- 分析数据库查询性能
- 检查缓存使用情况
- 审查前端资源加载优化
- 检查图片和媒体文件处理

### 步骤 6: 代码质量审查
- 检查代码规范和一致性
- 审查错误处理机制
- 检查日志记录
- 审查代码注释

### 步骤 7: 功能完整性审查
- 测试用户管理功能
- 测试内容管理功能
- 测试媒体管理功能
- 测试评论系统
- 测试权限系统

### 步骤 8: 测试和文档审查
- 检查测试覆盖率
- 审查 API 文档
- 检查部署文档

## 预期输出

生成一份详细的审查报告（`project_review_report.md`），包括：
1. 项目整体评分（满分10分）
2. 发现的问题列表（按严重程度分类：高、中、低）
3. 改进建议（每个问题对应具体的解决方案）
4. 优先级排序的任务清单
