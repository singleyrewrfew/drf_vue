核心业务流程

🔐 用户登录认证流程

1用户提交 用户名+密码 至 /api/auth/login/
2Django Auth 后端验证凭证 → 生成 JWT Token 对
3返回 Access Token + Refresh Token 给前端
4后续请求携带 Authorization: Bearer
5RBAC 中间件校验角色权限 → 放行/拒绝
6Token 过期时用 Refresh Token 自动轮换

📝 内容发布流程

1作者在编辑器创建内容 → 状态为 DRAFT
2选择分类(Category)和标签(Tag)关联
3上传媒体文件 → MD5去重 + 存储元数据到 Media 表
4点击发布 → ContentService.update_status(PUBLISHED)
5触发 Signal content_published → 清除相关缓存
6前端实时获取最新内容列表

🖼️ 媒体上传处理流程

1前端上传文件至 /api/media/upload/ (SSE进度回调)
2计算文件 MD5 哈希值，检查是否已存在
3命中: 引用计数 +1，直接返回已有记录
4未命中: 写入磁盘，创建 Media 记录
5若为视频 → 提交 Celery 任务异步转码+生成缩略图
6触发 mediauploaded 信号通知下游

💬 评论互动流程

1用户对内容发表评论 POST /api/comments/
2支持嵌套回复: parentcomment_id 指向父评论
3其他用户可点赞 POST /api/comments/{id}/like/
4CommentLike 记录（防重复点赞）
5软删除支持: is_deleted 标记而非物理删除
6触发 comment_created 信号（可扩展通知）

⚡ 数据缓存机制

1会话缓存: Redis 存储 Session，多实例共享登录态
2内容缓存: 热点文章/首页列表缓存至 Redis (TTL)
3Token黑名单: 登出/吊销的 Token 存入 Redis Set
4限流计数: API 请求频率限制 (IP/用户维度)
5SSE推送: 上传进度通过 Redis Pub/Sub 实时推送
6失效策略: Signal 触发 → 主动清除相关缓存 key

🌍 多环境配置

1通过环境变量 DJANGO_ENV 切换配置
2DevelopmentConfig: SQLite + DEBUG=True + 本地日志
3ProductionConfig: MySQL + Redis + Celery + 白名单
4.env.development / .env.production 分离敏感信息
5django-environ 统一加载，settings.py 自动验证
