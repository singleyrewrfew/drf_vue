## 二、🔴 严重问题（需立即修复）

### 🔴-1：API 实例配置硬编码重复
- **位置**：`api/index.js:47-57` vs `67-73`
- **问题**：`api` 和 `logoutApi` 两处完全重复 baseURL/timeout/headers 配置。后续修改超时时间等需同步修改两处。
- **建议**：提取公共配置对象：
  ```js
  const BASE_CONFIG = {
      baseURL: import.meta.env.VITE_API_BASE_URL,
      timeout: import.meta.env.VITE_API_TIMEOUT || 10000,
      headers: { 'Content-Type': 'application/json' },
  }
  const api = axios.create(BASE_CONFIG)
  const logoutApi = axios.create({ ...BASE_CONFIG })
  ```

### 🔴-2：TablePage 使用 alert() 作为反馈
- **位置**：`components/TablePage.vue:508`
- **问题**：批量操作使用原生 `alert()`，生产环境应替换为 Element Plus 组件。
- **建议**：改为 `ElMessage.success()` 或 `ElNotification`

### 🔴-3：CustomSelect 使用 Math.random() 生成 ID
- **位置**：`components/CustomSelect.vue:115`
- **问题**：`listboxId = Math.random().toString(36).slice(2)` — SSR 不安全且可能碰撞
- **建议**：改用 Vue 3.5+ 的 `useId()` 组合式函数或自增计数器

### 🔴-4：QuickActionCard !important 滥用
- **位置**：`components/QuickActionCard.vue:96`
- **问题**：`.quick-action-card:active { transform: ... !important; }`
- **建议**：通过提高选择器优先级解决

### 🔴-5：SelectWithCreate !important 强制覆盖
- **位置**：`components/SelectWithCreate.vue:49`
- **问题**：`!important` 覆盖 el-select 宽度
- **建议**：改用更高优先级选择器或 flex 布局

## 三、🟡 中等问题（影响可维护性）

### 🟡-1：Element Plus 全量引入 → 打包体积过大
- **位置**：`main.js:3`
- **影响**：全量引入约增加 **800KB+** 未使用组件代码
- **建议**：安装 `unplugin-vue-components` + `unplugin-auto-import` 按需引入

### 🟡-2：Composables 大量未被使用（死代码）
- **问题**：项目有 8 个 composable（usePagination/useFormSubmit/useDelete 等），但大部分 CRUD 视图仍手写分页/提交/删除逻辑
- **受影响视图**：category/Index.vue, category/Tags.vue, system/Role.vue, system/Permission.vue, user/Index.vue, dashboard/Media.vue, dashboard/Comments.vue, content/Index.vue
- **预估重复代码**：500+ 行
- **建议**：逐步迁移各页面到 composable 方案

### 🟡-3：CSS 文件职责重叠 + 选择器重复
- **文件**：`styles/main.css` vs `styles/element-override.css`
- **重复选择器**：`.el-button`, `.el-button--primary/success/danger`, `.el-card`, `.el-input__wrapper`, `.el-table`, `.el-pagination`, `.el-dialog`, `.el-tag--xxx` 共 9 类在两个文件中重复定义
- **建议**：合并为一个文件，明确职责分工

### 🟡-4：Login/Register 样式高度重复
- **文件**：`views/auth/Login.vue`(492行) vs `views/auth/Register.vue`(516行)
- **重复内容**：背景渐变、浮动形状动画、Logo 区域、密码 SVG 图标(×4)、按钮样式、页脚样式
- **预估**：~250 行重复 CSS
- **建议**：提取共享的 `AuthLayout.vue` 布局组件

### 🟡-5：密码 SVG 图标重复 6 次
- **位置**：Login.vue(1次)、Register.vue(3次)、Profile.vue(2次)
- **已有组件**：`PasswordInput.vue` 已实现此功能但未被复用
- **建议**：统一使用 PasswordInput 组件

### 🟡-6：TablePage.vue 超大组件 (745 行)
- **问题**：承担表格 + 分页 + 操作列 + 批量操作全部职责
- **建议**：拆分为 TableCore.vue + TablePagination.vue + TableActions.vue

### 🟡-7：VideoPlayer.vue 大型组件 (479 行)
- **问题**：视频播放器逻辑复杂度高
- **建议**：评估是否需要拆分子组件或提取 hooks

### 🟡-8：content/Form.vue 样式灾难 (390 行样式 + 130 行 !important)
- **问题**：大量 md-editor-v3 样式覆盖 + 硬编码亮暗色值
- **建议**：将覆盖样式提取到 `md-editor-override.css` 并使用 CSS 变量

### 🟡-9：403 拦截器中嵌套异步调用可能递归
- **位置**：`api/index.js:252-271`
- **问题**：响应拦截器中再次发起 API 请求(`fetchProfile`)，如果该请求也返回 403/401 可能引发竞态
- **建议**：添加防重入标记

### 🟡-10：超时时间硬编码
- **位置**：`api/index.js:49,68` 两处 `timeout: 10000`
- **建议**：提取为环境变量 `import.meta.env.VITE_API_TIMEOUT`

### 🟡-11：部分 API 函数 params 无默认值
- **文件**：`api/category.js`, `api/content.js`, `api/role.js`
- **问题**：`getCategories(params)` 等未给默认值 `params = {}`
- **建议**：统一添加 `= {}` 默认参数

### 🟡-12：health.js 缺少文档和注释
- **文件**：`api/health.js`（仅 4 行）
- **问题**：唯一没有 JSDoc 注释的 API 文件
- **建议**：补充标准注释模板

### 🟡-13：getUploadUrl 手动拼接 URL
- **位置**：`api/media.js:47`
- **问题**：`${VITE_API_BASE_URL}/media/` 直接拼接环境变量
- **建议**：使用 api 实例的 baseURL 配置

### 🟡-14：ImageUploader 固定宽度无响应式
- **位置**：`components/ImageUploader.vue:135`
- **问题**：`width: 320px` 硬编码
- **建议**：使用 `max-width` 或 CSS 变量

### 🟡-15：FormDialog v-bind="$attrs" 透传风险
- **位置**：`components/FormDialog.vue:17-18`
- **问题**：未知属性透传到 el-dialog 可能产生意外行为
- **建议**：明确列出允许透传的属性

## 四、🔵 轻微问题（锦上添花）

### 🔵-1：缺少 .env.production 配置文件
- **现状**：仅有 `.env.development`，无生产环境配置
- **风险**：部署时无法区分开发/生产环境 API 地址等
- **建议**：创建 `.env.production` 设置生产环境变量

### 🔵-2：npm audit 无法执行
- **原因**：当前 registry 设为 `npmmirror.com` 镜像，不支持 audit 请求
- **建议**：定期切换官方源检查安全漏洞，或使用 Snyk/Lighthouse CI 扫描

### 🔵-3：无 ESLint/Prettier 配置
- **现状**：项目中未发现 `.eslintrc.*` / `.prettierrc` / `.editorconfig`
- **风险**：团队协作时编码风格不一致
- **建议**：至少添加 `.editorconfig` 统一缩进和换行规范

### 🔵-4：QuickActionCard description prop 未使用
- **位置**：`components/QuickActionCard.vue:19` 定义了 `description` prop 但模板中从未渲染
- **建议**：删除或在模板中使用

### 🔵-5：StatCard 有 cursor:pointer 但无点击事件
- **位置**：`components/StatCard.vue:47`
- **建议**：移除 cursor:pointer 或添加 click handler + role="button"

### 🔵-6：ThumbnailStatus 死 CSS 类
- **位置**：`components/ThumbnailStatus.vue:34-38`
- **问题**：`.status-content` 在模板中未使用
- **建议**：清理

### 🔵-7：httpStatus.js 过度设计（276行常量）
- **问题**：定义了完整 HTTP 状态码，但实际使用极少（仅 api/index.js 引用了 UNAUTHORIZED/FORBIDDEN），大量常量为死代码
- **建议**：精简到只保留项目实际使用的状态码，或确认是否需要完整定义

### 🔵-8：AXIOS_ERROR_CODES 仅一个值且未被使用
- **位置**：`api/index.js:10-12`
- **问题**：定义了 TIMEOUT 常量但第 236 行直接使用了字符串字面量 `'ECONNABORTED'`
- **建议**：要么使用常量，要么移除

### 🔵-9：theme.ts 为唯一 .ts 文件
- **位置**：`stores/theme.ts`
- **问题**：与全项目纯 JS 风格不一致
- **建议**：重命名为 theme.js 或全面迁移 TypeScript

### 🔵-10：console.log 仅存在于 JSDoc 示例中
- **结果**：✅ 生产代码中无 console.log/debug 泄漏
- **说明**：仅在 useDialog.js/useConfirm.js/tokenRefresh.js 的 JSDoc 示例中出现，不影响运行时

### 🔵-11：v-html XSS 安全评估
- **位置**：`ActionButton.vue:10,15`
- **结论**：**低风险** — icon prop 经过 validator 校验（只接受预定义名称），iconSvg 从内置 icons 对象查找，非用户输入

### 🔵-12：localStorage 敏感数据存储
- **存储项**：access token、refresh token、用户信息（含权限列表）、主题偏好
- **风险评估**：token 存储在 localStorage 是 SPA 标准做法，但存在 XSS 窃取风险
- **建议**：如需更高安全性，可考虑 httpOnly cookie 存储 refreshToken

### 🔵-13：useMediaPreview dialogWidth 非响应式
- **位置**：`composables/useMediaPreview.js:53-55`
- **问题**：使用 window.innerWidth 初始化后不随窗口变化更新
- **建议**：监听 resize 事件或使用 useWindowSize()

### 🔵-14：自动保存策略缺陷
- **位置**：`views/content/Form.vue:319-323`
- **问题**：仅在 content.length > 100 时触发；每次发送完整表单数据而非 diff
- **建议**：移除长度阈值，改用 debounce + diff 比较
