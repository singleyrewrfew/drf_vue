**代码位置**：

| Composable | 已实现功能 | 未使用的视图页面 |
|------------|-----------|-----------------|
| `usePagination.js` | 分页状态管理、页码计算、翻页方法 | `category/Index.vue`、`category/Tags.vue`、`system/Permission.vue`、`system/Role.vue`、`user/Index.vue`、`dashboard/Comments.vue`、`dashboard/Media.vue`、`content/Index.vue` |
| `useFormSubmit.js` | 表单提交、验证、成功/错误处理 | 同上所有含表单的页面 |
| `useDelete.js` | 删除确认 + API 调用 + 成功提示 | 同上所有含删除功能的页面 |
| `useConfirm.js` | 通用确认对话框 | 同上 |

**重复模式示例**（每个页面都手写了几乎相同的代码）：

```javascript
// 以下模式在 8 个视图中重复出现
const pagination = reactive({ page: 1, pageSize: 10, total: 0 })
const fetchXxx = async () => {
  const offset = (pagination.page - 1) * pagination.pageSize
  const { data } = await api.getXxx({ offset, limit: pagination.pageSize })
  xxxList.value = data.results || data
  pagination.total = data.count || xxxList.value.length
}
const handlePageChange = (page) => { pagination.page = page; fetchXxx() }
```

**影响范围**：8+ 个视图文件，估计 500+ 行重复代码
**修复建议**：重构所有 CRUD 页面，使用 `usePagination`、`useFormSubmit`、`useDelete` 替换手写逻辑
**预估工作量**：3-4 天

### P1-4：路由守卫 Token 过期未验证

**问题描述**：`isLoggedIn()` 仅检查 `accessToken` 是否存在，**不验证 JWT 是否过期**。过期 token 仍会通过路由守卫，导致页面能进入但 API 请求全部 401。

**代码位置**：
- `stores/user.js:112` — `isLoggedIn()` 只检查 `!!accessToken.value`
- `router/index.js:142` — 路由守卫调用 `isLoggedIn()`
- `constants/authConfig.js:102-104` — `isValidToken` 只检查非空字符串

**修复建议**：解析 JWT payload 的 `exp` 字段，与当前时间比较：

```javascript
function isTokenExpired(token) {
  try {
    const payload = JSON.parse(atob(token.split('.')[1]))
    return payload.exp * 1000 < Date.now()
  } catch { return true }
}
```

**预估工作量**：2 小时

### P3-7：多标签页场景下 isLoggingOut 不同步

**代码位置**：`api/index.js:14`

**建议**：使用 `localStorage` 或 `BroadcastChannel` 跨标签页同步登出状态

### P3-8：useThumbnailSSE 自实现 SSE 不完整

**代码位置**：`composables/useThumbnailSSE.js:7-93`

**建议**：使用 `@microsoft/fetch-event-source` 或 `eventsource` 库

## 六、架构级系统性问题

### 1. 类型安全缺失

全项目使用纯 JavaScript，`defineProps` 只定义了最基本的类型（`String`、`Boolean`、`Number`），缺少精确的类型约束。

**建议**：
- 短期：为关键组件添加 `PropTypes` 校验（`validator` 函数）
- 长期：迁移 TypeScript，从 `api/` 和 `stores/` 层开始

---

### 2. 错误处理不一致

项目中错误提取逻辑分散，后端返回的错误字段名不统一：

| 使用的字段 | 代码位置 |
|-----------|----------|
| `error.response?.data?.message` | 多个视图 |
| `error.response?.data?.error` | 部分视图 |
| `error.response?.data?.detail` | 部分视图 |
| `error.response?.data?.username?.[0]` | user/Index.vue |
| `error.response?.data?.data?.message` | content/Form.vue |

**建议**：在 `api/index.js` 的 `extractErrorMessage` 中统一处理，所有视图只调用该函数

---

### 3. API 响应格式兼容代码重复

`data.results || data` 和 `data.count || xxxList.value.length` 在几乎所有 fetch 函数中重复出现。

**代码位置**：8+ 个视图页面的 fetch 函数

**建议**：在 API 层或 `usePagination` 中统一处理分页响应格式

---

### 4. 权限判断逻辑不统一

| 判断方式 | 代码位置 |
|----------|----------|
| `userStore.isAdmin()` | 大部分页面 |
| `userStore.user?.role_code === 'admin' \|\| userStore.user?.is_superuser` | `content/Form.vue:239` |
| `meta.requiresAdmin` 路由守卫 | `router/index.js:79` |

**建议**：统一使用 `userStore.isAdmin()`，删除各处重复的权限判断

---

### 5. 路由守卫逻辑缺口

**代码位置**：`router/index.js:153-157`

**问题描述**：访客页面且已登录时，如果用户不能访问后台，没有执行 logout 也没有跳转，已登录但无后台权限的用户停留在 guest 页面，既不被重定向也不被登出。

**建议**：对已登录但无后台权限的用户执行 logout 并重定向到登录页
